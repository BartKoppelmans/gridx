gridx/modules/filter/FilterBar.__ColumnDefinition (version 1.2)

Note:This is not a real constructor, but just a description of the type of object that should be passed as a parameter to some method(s), and/or the return value from some method(s). In other words, the type exists only for documentation purposes, and you cannot call new FilterBar.__ColumnDefinition().

Properties

dataType
Defined by gridx/modules/filter/FilterBar

Specify the data type of this column. Should be one of "string", "number", "date", "time", "boolean" and "enum". Case insensitive. Data type decides which conditions to use in the Filter Definition Dialog.

dataTypeArgs
Defined by gridx/modules/filter/FilterBar

Passing any other special config options for this column. For example, if the column is of type 'date', but the data in store is of string type, then a 'converter' function is needed here: dataTypeArgs: { useRawData: true, converter: function(v){ return dojo.date.locale.parse(v, {...}); } }

disabledConditions
Defined by gridx/modules/filter/FilterBar

If provided, all the listed conditions will not occur in the Filter Definition Dialog for future rules. But this does not influence existing filter rules. Default to be an empty array.

filterable
Defined by gridx/modules/filter/FilterBar

If FALSE, then this column should not occur in the Filter Definition Dialog for future rules. But this does not influence existing filter rules. Default to be TRUE.
