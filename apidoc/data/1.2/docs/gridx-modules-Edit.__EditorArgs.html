gridx/modules/Edit.__EditorArgs (version 1.2)

Arguments for the editor.

Note:This is not a real constructor, but just a description of the type of object that should be passed as a parameter to some method(s), and/or the return value from some method(s). In other words, the type exists only for documentation purposes, and you cannot call new Edit.__EditorArgs().

Property Summary

  • constraints If the editor widget has some constraints, it can be set here instead of in props.
  • props The properties to be used when creating the dijit in a editing cell.
  • useGridData Whether to feed the editor with grid data or store data.
  • valueField The property name of the editor used to take the data.

Method Summary

  • fromEditor(valueInEditor) By default when applying an editing cell, the value of the editor dijit will be retreived by get('value') and directly set back to the store.
  • toEditor(storeData, gridData) By default the dijit used in an editing cell will use store value.

Properties

constraints
Defined by gridx/modules/Edit

If the editor widget has some constraints, it can be set here instead of in props.

props
Defined by gridx/modules/Edit

The properties to be used when creating the dijit in a editing cell. Just like data-dojo-props for a widget.

useGridData
Defined by gridx/modules/Edit

Whether to feed the editor with grid data or store data. This property is only effective when toEditor is not provided.

valueField
Defined by gridx/modules/Edit

The property name of the editor used to take the data. In most cases it is "value", so editor.set('value', ...) can do the job.

Methods

fromEditor(valueInEditor)
Defined by gridx/modules/Edit

By default when applying an editing cell, the value of the editor dijit will be retreived by get('value') and directly set back to the store. If this can not meet the requirement, this getEditorValue function can be used to get a suitable value from editor.

ParameterTypeDescription

toEditor(storeData, gridData)
Defined by gridx/modules/Edit

By default the dijit used in an editing cell will use store value. If this default behavior can not meet the requirement (for example, store data is freely formatted date string, while the dijit is dijit.form.DateTextBox, which requires a Date object), this function can be used.
